# Лига триатлона · IX этап · Черкесск — 360° галерея

Лендинг со сферической фотогалереей на Photo Sphere Viewer 5.7 + Three.js.
Сборки нет — статические файлы.

## Два варианта

| Файл | Библиотеки | Когда использовать |
|---|---|---|
| `index-single.html` | вшиты внутрь | **просмотр с диска двойным кликом** |
| `index.html` | с CDN jsdelivr | публикация на сайт, по умолчанию |
| `index-offline.html` | из папки `vendor/` | публикация без зависимости от CDN |

**Важно:** `index.html` и `index-offline.html` двойным кликом не открываются —
браузер блокирует ES-модули на `file://` (ошибка CORS в консоли). Это не поломка,
так работает защита браузера. Для просмотра с диска используйте `index-single.html`,
для сайта — остальные. Подробнее в `КАК-ОТКРЫТЬ.txt`.

Оба протестированы в браузере, работают одинаково. Если берёте офлайн-вариант —
переименуйте его в `index.html`, а CDN-версию удалите. Папка `vendor/` весит ~1.5 МБ;
при использовании CDN-версии её можно удалить целиком.

Шрифты Google Fonts подключаются с CDN в обоих вариантах. Без интернета страница
не сломается — просто подставятся системные шрифты.

## Локальный запуск

Открыть двойным кликом **не получится** — ES-модули требуют HTTP.

```bash
cd <папка проекта>
python3 -m http.server 8000
# http://localhost:8000
```

## Как добавить панораму

1. Положите JPEG в `panoramas/`. Требования:
   - **строго 2:1** (equirectangular), например 6000x3000
   - до ~2 МБ, иначе долго грузится на мобильных

   ```bash
   convert original.jpg -resize 6000x3000 -quality 82 -strip panoramas/pano-03.jpg
   ```

2. Превью — кроп интересного участка, не вся сфера (иначе плитка выглядит искажённой):
   ```bash
   convert original.jpg -crop 2400x2400+2800+900 +repage -resize 640x \
     -quality 80 -strip panoramas/thumb-03.jpg
   ```

3. Добавьте объект в массив `PANORAMAS` (ищите комментарий «ДАННЫЕ ПАНОРАМ»)
   — **в обоих html-файлах**, если храните оба:

```js
{
  id: 'p3',
  src:   'panoramas/pano-03.jpg',
  thumb: 'panoramas/thumb-03.jpg',
  title: 'Велоэтап',
  sub:   'Разворотная точка на трассе',
  tag:   'track',          // start | team | track
  tagLabel: 'Трасса',
  startYaw: 180            // см. ниже
}
```

Счётчик в шапке и нумерация плиток обновляются сами.

## startYaw — стартовый угол

**Зачем.** Шов склейки у Insta360 всегда приходится на край кадра, то есть на 0°.
Если открыть панораму со стартовым углом 0, зритель первым делом увидит склейку.
Поэтому у каждой панорамы задаётся свой угол — примерно напротив шва.

Текущие значения подобраны по центру сюжета:

| Панорама | startYaw | Примечание |
|---|---|---|
| Судейский створ | 18 град. | развёрнуто на 180 по запросу; шов попадает в кадр |
| Команда этапа | 50 град. | развёрнуто на 180 по запросу; шов вне кадра |

Прежние значения, при которых шов не был виден ни в одной панораме — **198** и **230**.
Если шов на первой панораме будет мешать, верните их.

**Как подобрать для новой панорамы.** Откройте её на странице, покрутите до нужного
вида и посмотрите угол в консоли браузера:

```js
Math.round(viewer.getPosition().yaw * 180 / Math.PI)
```

Либо начните со 180 и подвиньте на глаз. Автоматический подбор «по количеству
деталей» работает плохо: яркие баннеры перетягивают расчёт на себя — по второй
панораме он предлагал 248, хотя группа при этом уезжала за левый край.

## Автоповорот

Плагин `@photo-sphere-viewer/autorotate-plugin`.

- **В шапке** — крутится сам, останавливается при перетаскивании и возобновляется
  через 1.5 с простоя. При `prefers-reduced-motion` не запускается.
- **В лайтбоксе** — только по кнопке в панели. Когда рассматриваешь кадр,
  самопроизвольное вращение мешает.

Скорость меняется в `autorotateSpeed` (сейчас `0.6rpm` и `0.5rpm`).

## Управление

| Действие | Как |
|---|---|
| Осмотреться | перетаскивание мышью / пальцем |
| Чувствительность мыши | `moveSpeed: 2` в коде (1 = слишком вяло) |
| Зум в шапке | **Ctrl + колесо** (обычное колесо скроллит страницу) |
| Зум в лайтбоксе | колесо или кнопки |
| Листать панорамы | стрелки влево/вправо или кнопки |
| Закрыть | Esc |

Колесо в шапке специально повешено на Ctrl — иначе панорама перехватывает скролл
и страницу невозможно пролистать вниз.

## Заглушки

`const PLACEHOLDER_SLOTS = 2;` — количество пунктирных «мест под кадр» в конце сетки.
Поставьте `0`, когда галерея заполнена.

## Фильтры

Категории заданы в HTML в блоке `.filters` через `data-filter`. Чтобы добавить —
допишите кнопку и используйте то же значение в поле `tag` у панорам.

## Публикация

**GitHub Pages** — как lapin-lab.ru: залить репозиторий, включить Pages, ветка `main`, папка `/`.

Панорамы весят ~2 МБ каждая. Если их станет больше 15–20, лучше вынести `panoramas/`
на объектное хранилище (Yandex Object Storage) и указать полные URL в `src`.
